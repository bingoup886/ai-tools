## 💡 最佳实践建议

### 日常开发

1. **使用 preview 分支进行开发**
   ```bash
   git checkout preview
   # 进行改动
   git push origin preview
   ```

2. **定期更新 CHANGELOG**
   - 不要等到发布时才更新
   - 及时记录重要变更

3. **遵循提交规范**
   ```bash
   git commit -m "feat: 功能描述"
   git commit -m "fix: 修复描述"
   git commit -m "docs: 文档描述"
   ```

### 版本发布

1. **使用语义版本控制**
   - MAJOR.MINOR.PATCH
   - 根据变更类型递增版本号

2. **创建 git 标签**
   ```bash
   git tag v1.1.0
   git push origin v1.1.0
   ```

3. **更新 CHANGELOG**
   - 将 [Unreleased] 改为版本号
   - 添加发布日期
